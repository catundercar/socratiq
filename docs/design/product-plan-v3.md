## 1. 产品定位

### 一句话定位

> Socratiq 是一个面向技术自学者的 AI 学习导师，它能把 YouTube 视频和 PDF 等外部资料自动转成个性化学习路径，并在练习与项目中持续纠偏。

### 定位拆解

| 维度 | 内容 |
|------|------|
| **目标用户** | 技术自学者（学 AI / 编程 / 系统设计 / 数据科学的人） |
| **输入** | 用户自己找到的外部学习资料（YouTube 视频、PDF 论文/书籍等） |
| **输出** | 个性化学习路径 + 针对性练习 + 持续纠偏 |
| **差异** | 不是资料摘要（NotebookLM），不是平台内教学（Khanmigo），而是**把任意资料变成学习推进器** |

### 产品原则

1. **导师，不是工具**——系统的目标不是"回答问题"，而是"推进学习"
2. **知识库是基础设施层，导师 + 画像是价值层**——摄入内容不是目的，因材施教才是
3. **先做窄做深，再做宽做通用**——V1 只服务技术自学者，只支持有限来源
4. **用户愿意把下一份资料继续交给你学，是最早的 PMF 信号**

## 2. 目标用户与 JTBD

### 用户分层

| 层级 | 画像 | 特征 | 优先级 |
|------|------|------|--------|
| **核心用户（V1）** | 技术自学者 | 学 AI/编程/系统设计；有明确学习目标；已有资料但不会系统学；愿意做练习而非只看摘要 | 🔴 现在 |
| **次级用户（V2）** | 职业进阶者 | PM/数据分析师/运营；通过公开资料做职业转型；需要证书备考辅助 | 🟡 6个月后 |
| **远期用户（V3）** | 泛学习者 / 学校场景 | 通用学科学习；教师教学辅助；家长/学生 | 🟢 12个月后 |

### 核心用户 JTBD

> **当我**收集了一堆高质量但零散的学习资料时，
> **我需要**一个系统帮我整理学习顺序、解释难点、布置练习，并根据我的错误调整后续学习，
> **这样**我才能真的学会，而不是只是"看过"。

### 核心用户的痛点清单

| 痛点 | 现有方案 | 为什么不够好 |
|------|---------|------------|
| 收集了很多资料但不知道从哪开始 | 自己列计划 | 容易高估自己，计划三天就废 |
| 看完视频/论文感觉"懂了"但其实没懂 | 做笔记 | 缺乏检验机制，虚假掌握感 |
| 遇到卡点不知道是缺了什么前置知识 | 搜 Google / 问 ChatGPT | ChatGPT 不知道你的上下文，每次从零开始 |
| 学了后面忘了前面 | 自己安排复习 | 没有数据支持，全靠自觉 |
| 资料之间的知识有重叠和关联，自己串不起来 | 无有效方案 | 这正是 Socratiq 的核心差异 |

### 核心用户的验证假设

需要在 MVP 阶段证伪或证实的关键假设：

| # | 假设 | 验证方式 |
|---|------|----------|
| H1 | 用户愿意把外部资料导入系统学习，而非只是读摘要 | 导入后是否进入学习流程（而非只看生成的大纲） |
| H2 | AI 生成的学习路径比用户自己安排的更有效 | 路径完成率 vs 自定义路径完成率 |
| H3 | 基于练习反馈的纠偏能让用户感知到"比 ChatGPT 好" | 用户定性反馈 + 二次导入率 |
| H4 | 用户会因为"系统记得我"而持续回来 | D7/D30 留存 vs 无画像对照 |

## 3. 竞品空位分析

### 市场格局：四个象限

```
                    内容来源开放
                        ↑
                        |
    NotebookLM          |         Socratiq
    (理解资料但不教你)    |         (把资料变成学习推进)
                        |
  ←—————— 被动应答 ——————+—————— 主动教学 ——————→
                        |
    ChatGPT / Claude    |         Khanmigo / Duolingo
    (什么都能聊但无记忆)  |         (会教但锁在自有内容)
                        |
                        ↓
                    内容来源封闭
```

**Socratiq 要占的空位**：右上角——内容来源开放 × 主动教学。

当前没有产品同时做到这两点。

### 竞品空位而非竞品碾压

| 竞品 | 它占据的位置 | 它的强项 | 它留下的空位 |
|------|-----------|---------|------------|
| **NotebookLM** | 资料理解 | 多源摄入、grounded output、Audio Overview | 无学习记忆、无练习、无纠偏——理解不等于学会 |
| **Khanmigo** | 平台内教学 | 苏格拉底式引导、$4 低价、70 万学生验证 | 锁死在 Khan 内容库——用户自己找的资料无法用 |
| **Duolingo** | 习惯留存 | 游戏化驱动的 DAU/MAU 39.6%、$10 亿收入 | 单一领域（语言）——且教学质量长期被诟病 |
| **Coursera Coach** | 课程内陪学 | 嵌入 7,000+ 课程、提升 9.5% 通过率 | 只能在 Coursera 平台内使用 |
| **ChatGPT / Claude** | 通用问答 | 什么都能聊 | 无学习记忆——每次从零开始，无法推进学习 |

### 我们从竞品学到的三件事

1. **从 Duolingo 学留存**：游戏化不是锦上添花，它是教育产品的首要生存问题（教育 App Day 30 留存平均 2.1%）
2. **从 Khanmigo 学教学**：苏格拉底式引导（不直接给答案）是经过 70 万学生验证的最有效 AI 教学方式
3. **从 Quizlet Q-Chat 学教训**：纯 AI 聊天包装层不构成护城河——2025 年 6 月关闭

## 4. 核心价值主张

### 用户能感知到的三个"比 ChatGPT 好"的瞬间

这三个瞬间是 Socratiq 存在的理由。如果用户在使用中感受不到这三点的任何一点，产品就失败了。

**瞬间 1：「它比我自己整理资料更快」**

用户粘贴一个 YouTube 链接或上传一个 PDF，3 分钟后看到：
- 按难度排序的学习路径
- 每个章节的核心概念和前置知识
- 基于用户当前水平的"今天建议从这里开始"

**瞬间 2：「它真的知道我哪里不会」**

用户在练习中答错了一道关于 attention 机制的题，系统不是简单说"答错了"，而是：
- 识别出用户对矩阵乘法的理解有缺口
- 回溯到前置知识，用用户更容易接受的方式重新讲解
- 下一组练习自动聚焦在这个薄弱点

**瞬间 3：「它不只是解释，而是推着我往前走」**

用户问完一个问题后，系统不是给个答案就结束，而是：
- 追问一个引导思考的问题（苏格拉底式）
- 给出下一步行动建议："理解了这个概念后，你可以尝试这个练习"
- 把这次学习推进记入进度，更新学习路径

### 护城河（不是 V1 就有，但必须从 V1 开始积累）

| 护城河 | 怎么积累 | 为什么竞品难复制 |
|--------|---------|----------------|
| 长期学生画像数据 | 每次交互更新画像 | 用户越用越准——迁移成本随时间增长 |
| 内容网络效应 | 用户导入的资料 → 概念图谱 → 惠及其他学习者 | 越多人用，知识关联越丰富 |
| 社区学习路径 | 用户分享自己的学习路径 | UGC 飞轮——NotebookLM 和 Khanmigo 都没有 |

## 5. 核心使用流程

### 流程 A：首日导入学习（最关键——决定用户是否留下）

```
第 1 步：导入资料
  用户粘贴 YouTube 链接（或上传 PDF）
  ↓
第 2 步：选择学习目标
  · 快速了解大意
  · 系统掌握核心概念
  · 实战应用（做项目/写代码）
  ↓
第 3 步：系统输出
  · 学习路径（章节列表 + 难度标注）
  · 预估总学习时长
  · 基于初始画像的水平判断
  · "今天建议先学这一节"
  ↓
第 4 步：开始第一节
  · 看视频片段 / 读 PDF 段落
  · 教案与内容联动展示
  ↓
第 5 步：第一次交互
  · 完成一个微练习（1-2 题）
  · 或对导师提一个问题
  ↓
第 6 步：第一次个性化反馈
  · 导师基于回答调整后续建议
  · "你对 X 的理解很好，但 Y 这块我们需要多练习"
  · 更新画像
```

**首日体验的硬指标**：用户从导入到收到第一次个性化反馈，**不超过 10 分钟**。

### 流程 B：持续学习（日常使用）

```
打开系统 → 看到"今日建议"
  ↓
继续当前学习路径的下一节
  ↓
遇到不懂的 → 向导师提问 → 导师用苏格拉底式引导
  ↓
完成本节练习 → 系统判分 + 错误分析
  ↓
画像更新 → 下一节内容自动调整难度和深度
  ↓
（如果有到期复习内容）→ 导师提醒复习 → 完成间隔重复
```

### 流程 C：项目实战（进阶使用，Phase 3+）

```
完成一个学习阶段 → 系统推荐综合项目
  ↓
项目需求文档 + 任务拆解
  ↓
用户写代码 → 提交 → 导师 Review
  ↓
识别实际薄弱点 → 回流到学习路径
```

## 6. MVP 范围与不做项

### MVP 定义：最短价值闭环

**用户导入一个 YouTube 视频 → 系统生成学习路径 → 用户边学边问 → 系统给出针对性练习 → 根据练习结果调整下一步内容。**

就这五件事。

### MVP 做什么（P0）

| 功能 | 为什么是 P0 | 验证什么假设 |
|------|-----------|------------|
| YouTube 字幕提取 + LLM 内容分析 | 没有内容摄入，后面全部不成立 | H1: 用户愿意导入资料 |
| 学习路径生成 | 核心价值的第一个感知瞬间 | H2: AI 路径比自己安排更有效 |
| 导师 Agent（苏格拉底式引导） | 核心差异化——"不只是回答，而是推进" | H3: 比直接问 ChatGPT 好 |
| 学生画像 v1（JSON + 对话推断更新） | 让导师"记得你"——"贴身私教"的基础 | H4: 记忆带来留存 |
| 基础 RAG（pgvector 内容检索） | 导师回答必须基于实际内容而非幻觉 | — |
| 微练习生成（每节 2-3 题） | 检验学习效果，不能只"看过" | H2, H3 |
| 冷启动诊断（5-10 题自适应评估） | 解决新用户无画像的体验断裂 | — |
| 基础间隔重复 | 最低限度解决"忘了"的问题 | — |
| 前端：课程创建 + 视频播放 + 导师对话（SSE 流式） | 基础可用界面 | — |

### MVP 不做什么

| 功能 | 为什么不做 | 什么时候做 |
|------|----------|----------|
| Bilibili / PDF / Markdown / URL 支持 | 先跑通一个来源 | Phase 2 |
| 主动探索引擎 | 中期差异化能力，不是首日价值 | Phase 2 |
| 知识图谱可视化 | 锦上添花，不是核心价值 | Phase 2-3 |
| 多 Agent 架构 | 单 Agent 足以验证核心假设 | Phase 3 |
| 游戏化（连续天数、排行榜） | 重要但不是 V1 的核心证明题 | Phase 2（但紧接 MVP 后立刻做） |
| Lab + 代码沙箱 | 价值高但工程量大 | Phase 3 |
| 社交功能 / 学习小组 | 增长功能，不是核心价值 | Phase 3-4 |
| B2B / Teams | 先验证个人用户 PMF | Phase 3-4 |
| 课程市场 | 平台化是远期愿景 | Phase 4 |
| 移动端 | Web 先行 | Phase 4 |
| 跨源关联 / 概念图谱 | 需要多源数据积累后才有意义 | Phase 2-3 |

### MVP 时间线：6 周（缩短自原 9 周）

| 周 | 交付物 |
|---|--------|
| W1 | 项目骨架搭建（FastAPI + Next.js + DB + Claude API 连通） |
| W2 | YouTube 字幕提取 + LLM 内容分析 + 学习路径生成 |
| W3 | 导师 Agent 核心循环 + 学生画像 v1 + 基础 RAG |
| W4 | 微练习生成 + 判分 + 画像更新 + 间隔重复 |
| W5 | 冷启动诊断 + 前端整合（视频播放 + 对话 + 练习） |
| W6 | 端到端测试 + bug 修复 + 10 人内测 |

## 7. 指标体系

### 北极星指标

> **用户是否愿意把下一份资料继续交给 Socratiq 学。**
>
> 量化为：7 天内二次导入率（导入第二个学习资料的用户占比）

### 指标金字塔

```
                    ┌─────────────────┐
                    │   北极星指标     │
                    │ 7日二次导入率    │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ↓              ↓              ↓
      ┌──────────────┐ ┌──────────┐ ┌──────────────┐
      │   激活指标    │ │ 学习指标  │ │  留存指标    │
      └──────────────┘ └──────────┘ └──────────────┘
```

### 分层指标

| 层级 | 指标 | 目标 | 含义 |
|------|------|------|------|
| **激活** | 首次导入→10分钟内完成"学+问+练"闭环的比例 | >60% | 用户是否快速感知到价值 |
| **学习推进** | 首个学习路径第一章完成率 | >50% | 用户是否真的在学 |
| **学习效果** | 同类题二次正确率提升 | >20pp | 用户是否真的学会了 |
| **留存** | Day 7 留存 | >40% | 用户是否回来 |
| **留存** | Day 30 留存 | >5% | 用户是否持续使用 |
| **深度** | 7 天内继续同一学习目标（vs 切到别的内容）| >60% | 用户是否在系统性学习 |
| **转化** | 免费→付费转化率 | >5% | 用户是否认为值得付费 |

### 反指标（需要警惕的信号）

| 信号 | 含义 | 行动 |
|------|------|------|
| 用户导入后只看大纲不进入学习 | 被当成了 NotebookLM，不是导师 | 强化"开始学习"的引导 |
| 用户只和导师聊天不做练习 | 被当成了 ChatGPT，不是学习系统 | 在对话中更积极嵌入练习 |
| 导入多个资料但每个只学一点 | 用户在探索而非学习 | 优化路径推荐的准确性 |

## 7.X 视频内容接入策略（YouTube / B站）

### 需求背景

Socratiq 的核心流程始于用户导入外部视频。视频接入的质量直接决定首日体验——如果播放器加载慢、嵌入失败、或进度无法追踪，整个学习闭环在第一步就断裂了。

视频接入的目标不是替代原始内容平台，而是在**官方播放器能力之上叠加学习增强层**，把视频从"被动观看内容"转化为"可引导、可练习、可追踪的学习单元"。

### 用户价值

| 场景 | 没有 Socratiq | 有 Socratiq |
|------|-----------------|----------------|
| 看 3.5 小时的 Karpathy 视频 | 看完觉得"懂了"，一周后忘光 | 自动分章节，每章后微练习验证，薄弱点标记 |
| 视频里提到一个不懂的概念 | 暂停→切到 Google 搜→忘了刚才看到哪 | 在当前时间点直接问导师，不离开学习流 |
| 想回顾某个知识点 | 拖进度条来回找 | 章节化导航+高频回放区域自动标记为难点 |

### 产品原则

1. **官方能力优先**：仅基于 YouTube / B站官方嵌入能力完成播放，不自行抓流、转存、转码、去广告或重托管视频内容
2. **学习层与内容层解耦**：视频平台负责内容分发与播放，Socratiq 负责学习路径编排、时间点问答、概念解释、章节化导航、微练习、进度记录、错题回流
3. **可降级**：当视频因平台限制无法内嵌时，不阻断学习——自动降级为跳转原平台播放，保留笔记、问答、练习与进度跟踪能力

### 方案设计

#### YouTube 接入

- **方式**：V1 采用 YouTube 官方 iframe 嵌入播放器 / IFrame Player API
- **支持能力**：单视频嵌入、指定时间点播放、播放进度同步、基于时间戳的问答、章节跳转、观看行为埋点（开始/暂停/完成/重复观看片段）
- **尺寸要求**：最小 200×200，建议 16:9 场景至少 480×270
- **已知限制**：上传者关闭嵌入、年龄限制、私有内容时无法在第三方页面播放；嵌入受 YouTube API Terms of Service 与 Developer Policies 约束；需正确传递 HTTP Referer 否则可能触发 error 153；视频本身设置的广告在嵌入时仍可能展示

#### B站接入

- **方式**：V1 采用 B站官方站外播放器（`player.bilibili.com/player.html`），支持 bvid/aid/cid/seasonId/episodeId 等参数
- **支持能力**：单视频嵌入、多 P 切换、指定时间点播放、弹幕开关控制、播放进度同步、基于时间点的问答/练习/笔记联动
- **尺寸**：官方建议业务侧自行控制，最佳比例 16:9

#### 统一体验：Video Learning Shell

为避免 YouTube 与 B站在产品层形成割裂，抽象统一的视频学习容器：

**界面布局**：
- 左侧：视频播放器（YouTube 或 B站官方播放器）
- 右侧：导师问答、概念解释、学习笔记
- 下方：章节导航、微练习、学习进度
- 顶部：当前学习目标 / 当前章节 / 预计剩余时间

**统一事件模型**（无论底层是哪个平台）：
- `onPlay` / `onPause` / `onSeek` / `onSegmentComplete` / `onReplaySegment` / `onVideoEnd`
- 驱动：时间点问答、章节完成判定、观看进度记录、自动弹出微练习、学习状态更新

**学习增强能力**：
- 时间戳问题绑定：用户提问时自动附带当前时间点上下文
- 章节化学习：将长视频切分为可学习的小单元
- 微练习插入：观看一段后立即验证理解
- 错误回流：根据答题情况推荐复习片段
- 重看检测：识别高频回放区域，标记为潜在难点

### 边界限制

| 限制 | 说明 |
|------|------|
| 不自建视频播放器 | 全部依赖官方嵌入能力 |
| 不下载/转存/转码 | 不碰视频流 |
| 不去广告/去品牌 | 尊重平台规则 |
| 不批量抓取 | 仅按用户请求逐个处理 |

### 异常与降级策略

| 异常场景 | 触发条件 | 处理方式 |
|----------|---------|---------|
| **不可嵌入** | YouTube 禁止嵌入 / 年龄限制 / 私有内容 / B站嵌入异常 | 显示"前往原平台观看" + 保留学习侧能力（笔记/问答/练习/进度） |
| **播放受限** | 网络异常 / 播放器加载失败 / 平台风控 / 引用来源限制 | 自动重试 → 提示刷新 → 提供原平台链接 → 学习记录不丢失 |

### MVP 范围

**V1 做**：YouTube 官方嵌入、B站官方嵌入、指定时间点跳转、观看进度记录、时间点问答、微练习弹出、不可嵌入时的降级跳转

**V1 不做**：自建播放器、下载/转存、视频转码、去广告/去品牌、批量抓取、复杂直播互动

### 埋点指标

| 指标 | 含义 | 目标 |
|------|------|------|
| 嵌入成功率 | 用户导入的视频能正常内嵌播放的比例 | >90% |
| 章节完成率 | 每个章节被完整观看的比例 | >60% |
| 微练习触发率 | 章节结束后用户实际做了微练习的比例 | >50% |
| 高频回放区域→难点标记准确率 | 被标记的难点是否确实是用户不理解的 | 定性验证 |
| 降级跳转后继续学习的比例 | 跳转到原平台后用户是否回来做练习 | >30% |
| 时间点问答使用率 | 在观看过程中使用导师提问的用户比例 | >40% |

### 一句话决策

> V1 支持 YouTube 与 B站官方嵌入播放，平台聚焦构建"学习增强层"，不介入视频内容分发。

## 8. 定价假设与验证路径

### 用户为什么会付费？

不是为"AI 能聊天"付费（ChatGPT 已经免费了）。
而是为以下体验付费：

| 付费触发点 | 免费版体验 | 付费版体验 |
|-----------|----------|----------|
| 更多资料源 | 3 个 YouTube 视频 | 无限视频 + PDF + 更多 |
| 更深的个性化 | 基础画像 | 深度画像 + 精准纠偏 |
| 更强的练习 | 每节 2 题 | 无限练习 + 代码 Lab |
| 持续学习推进 | 有限对话 | 无限对话 + 间隔重复提醒 |
| 长期记忆 | 基础记忆 | 完整记忆——三个月后仍记得你 |

### 定价方案（假设，需验证）

| 层级 | 价格 | 核心限制 |
|------|------|---------|
| **免费** | $0 | 3 个内容源、每天 10 次 AI 对话、基础画像 |
| **Pro** | $9.99/月（$79/年） | 无限量、深度画像、间隔重复、优先模型 |

B2B 定价暂不规划，等个人用户 PMF 验证后再考虑。

### 先验证哪类用户愿意付费

按照付费意愿排序：

1. **准备转岗的开发者**（学 AI/ML）——目标明确、时间敏感、已有付费学习习惯
2. **在职工程师系统性提升**（系统设计、新框架）——有雇主报销可能
3. **CS 在校生补充学习**——价格敏感但需求强烈

### LLM 成本约束

- 目标：单位用户 AI 成本 < $1.50/月
- 方案：70% 请求用预算模型（Haiku 级）、20% Sonnet、10% Opus
- $9.99 订阅下毛利率目标 >70%

## 9. 产品路线图

### Phase 1: MVP — 证明"把资料变成学习"有人要（6 周）

**做**：YouTube 摄入 → 路径生成 → 导师对话 → 微练习 → 画像 v1
**证明**：H1-H4 四个核心假设
**成功标准**：50-100 beta 用户，7 日二次导入率 >30%，D7 留存 >40%

### Phase 2: 留存 + 内容扩展 — 证明"用户会回来"（月 2-5）

**做**：
- 游戏化 v1（连续天数、每日微目标、进度可视化）——**最高优先级**
- PDF + Markdown 内容源
- 画像 v2（情节记忆 + 元认知记忆）
- 主动推送（间隔重复提醒 + 基于薄弱点的内容推荐）
- 知识图谱 v1

**证明**：游戏化是否显著提升 D30 留存
**成功标准**：1,000 活跃用户，D30 >5%，DAU/MAU >15%

### Phase 3: 深度 + 变现 — 证明"用户愿意付费"（月 5-8）

**做**：
- 付费版上线
- Lab + 代码沙箱
- 主动探索引擎（新内容消化、跨源关联、定时巡检）
- Bilibili + URL 内容源
- 实战项目设计 + 导师 Review

**证明**：核心用户是否为"持续陪学"付费
**成功标准**：$10K MRR，付费转化 >5%

### Phase 4: 平台化 — 网络效应（月 8-18）

**做**：移动端、社交功能、课程市场、B2B、API/LMS 集成
**证明**：平台网络效应是否出现
**成功标准**：$100K MRR，50,000 用户

## 10. 风险与应对

### 致命风险

| 风险 | 为什么致命 | 应对 |
|------|----------|------|
| **Day 30 留存 <3%** | 教育 App 平均只有 2.1%，用户来了就走 | Phase 2 紧接 MVP 做游戏化；参考 Duolingo |
| **用户把系统当 NotebookLM 用**（只看摘要不学习） | 核心价值没有被感知 | 强化首日"学+问+练"闭环引导 |
| **Google/OpenAI 直接做教育 AI** | 巨头进场碾压 | 护城河：长期画像数据 + 内容网络 + 社区 |

### 高风险

| 风险 | 应对 |
|------|------|
| 冷启动体验差 | 自适应诊断评估 + 降级体验独立好用 |
| LLM 幻觉导致错误教学 | 所有回答基于 RAG 检索 + 来源引用 |
| 画像推断不准 | 渐进式置信（1 次记录→3 次可能→5 次确认→主动验证） |
| LLM API 成本超预期 | 模型路由（70% 预算模型）+ 缓存 |
| 内容摄入质量不稳定 | 质量评分 + fallback（Whisper 转录） |

### 监管风险

- EU AI Act 将教育 AI 列为高风险系统（2026 年 8 月生效）
- 美国 COPPA 更新规则（2026 年 4 月生效）
- **应对**：从 Day 1 做 privacy-first 架构，数据本地化选项，审计轨迹

## 附录：技术方案摘要

> 详细技术设计见《Socratiq 系统设计文档》（独立文档）。
> 本节只保留与产品体验直接相关的技术约束。

### 技术栈

- **前端**：Next.js 14+ / React / TypeScript / Tailwind / shadcn/ui / Monaco Editor
- **后端**：Python / FastAPI / Celery + Redis
- **数据**：PostgreSQL + pgvector
- **AI**：Claude API（Anthropic）
- **部署**：Docker Compose

### 体验约束

| 约束 | 对产品的影响 | 缓解 |
|------|------------|------|
| LLM 响应延迟 1-5 秒 | 用户等待感 | SSE 流式输出，逐字显示 |
| 上下文窗口有限 | 不能把所有记忆塞进去 | 分层记忆架构 + 精准检索 |
| YouTube 字幕可能缺失 | 部分视频无法处理 | Whisper 转录 fallback |
| PDF 双栏/复杂排版 | 提取质量不稳定 | pymupdf + 质量评分 + 人工回退 |

### 记忆体系（简述）

Agent 的记忆分为六层，每层解决不同问题：

| 层 | 内容 | 特点 |
|---|------|------|
| **工作记忆** | 当前会话对话 | 短期，每次会话重置 |
| **画像记忆** | 学生认知模型（能力、风格、薄弱点） | 长期，持续更新 |
| **情节记忆** | 关键学习事件（顿悟、卡点、有效讲解） | 长期，按重要性衰减 |
| **内容记忆** | 导师读过/教过的内容 | 长期 |
| **进度记忆** | 每个概念的掌握度 + 间隔重复调度 | 长期，实时更新 |
| **元认知记忆** | 哪种教学策略对这个学生有效 | 长期 |

Agent 在每次推理时不加载全部记忆，而是根据当前上下文精准检索相关记忆注入 system prompt。
