## 1. 产品定位

### 一句话定位

**Socratiq — 把任意 YouTube / Bilibili 视频变成带 AI 导师的互动课程。自托管，支持多 LLM 后端。**

### 三层卖点

1. **Hook（让人停下来看）**：把任意 YouTube / Bilibili 视频变成带 AI 导师的互动课程
2. **价值（让人觉得值得试）**：自托管、支持 Ollama/Claude/GPT/DeepSeek，你的数据你做主
3. **深度（让人觉得有技术含量）**：5 层记忆系统 + 苏格拉底教学法 + 自适应学习路径

### 目标用户

技术领域的开发者和自学者，会用 Docker，愿意自己配 API key 或跑 Ollama。

### 开源版目标

1. GitHub star 和社区影响力
2. 建立技术口碑，为后续 SaaS 版积累用户认知
3. 吸引贡献者参与

### 成功标准

- 首月 GitHub star > 500
- README → clone → 跑通的转化率高（靠 issue 数量和反馈判断）
- 至少 5 个外部 contributor 提交 PR

## 2. 开源版 vs SaaS 版边界

| | 开源版（本期） | SaaS 版（后续） |
|---|---|---|
| 部署 | 自托管 docker compose | 托管服务 |
| 用户 | 单用户 | 多用户 + 认证 |
| LLM | 自配（Ollama / API key） | 平台提供，按量计费 |
| 内容源 | YouTube + Bilibili | + PDF / URL / Markdown |
| 游戏化 | 无 | Streaks / 成就 / 积分 |
| 付费 | 免费 | Freemium |
| 代码沙箱 | 下载 ZIP | 在线执行 |
| Profile 治理 | 无 | 用户可查看/纠正 AI 理解 |
| 高级分析 | 无 | 学习统计仪表盘 |
| 开源协议 | AGPL-3.0 | 商业许可 |

## 3. 功能优先级

### P0 — 没有就不是这个产品

| 功能 | 产品价值 |
|---|---|
| YouTube + Bilibili 视频导入 | 产品入口，用户的第一个动作 |
| AI 课程生成 | 核心差异化 — "视频变课程"的魔法时刻 |
| 结构化课程学习 | 让用户有路径感，不是散装内容 |
| Mentor AI 对话（SSE 流式） | 产品灵魂 — 有导师的感觉 vs 自己看视频 |
| 苏格拉底教学模式 | 核心教学方法论，竞品没有的深度 |
| 练习生成与评估 | 闭环学习 — 不只是"看"，还要"做" |
| 5 层记忆系统 | 最大技术护城河 — AI 记住你，越用越懂你 |
| 间隔复习（SRS） | 科学记忆，长期留存的基础 |
| 多 LLM 后端支持 | 开源用户最在意：Ollama 本地跑 = 零成本 + 隐私 |

### P1 — 显著提升产品价值

| 功能 | 产品价值 |
|---|---|
| 引用卡片 | 信任层 — 让用户看到答案有据可查，不是幻觉 |
| 冷启动诊断 | 解决"AI 不了解我"的初始体验问题 |
| 知识图谱可视化 | Wow factor — README 截图吸引力，体现技术深度 |
| 视频内嵌播放 + 时间戳联动 | 学习体验的连贯性 — 看课程同时能回看原视频 |

### P2 — SaaS 阶段再完善

| 功能 | 产品价值 |
|---|---|
| Profile 治理 UI | 有价值但单用户场景紧迫性低 |
| 交互模式可见性 | 体验优化，非核心 |
| RAG 离线评估管线 | 开发者自用不需要系统化评估 |
| PDF / URL / Markdown 源 | 已有代码，但不作为 MVP 打磨重点 |

## 5. 开发者体验（DX）

### 一键启动

```bash
git clone https://github.com/xxx/socratiq.git
cd socratiq
cp .env.example .env  # 配一个 LLM 后端就行
docker compose up
# 打开 http://localhost:3000
```

**要求：5 分钟内跑通，零报错。**

### 配置简化

- 合理的默认值 — DB/Redis/JWT 等基础设施全部有默认值
- 最小配置 — 用户只需决定：用 Ollama（零配置）还是填一个 API key
- Setup 页面 — 首次打开引导配置 LLM

## 6. 测试与质量保障

### 核心学习闭环测试

| 测试场景 | 验证点 |
|---|---|
| YouTube 视频导入 | 粘贴 URL → 提取字幕 → 内容分析 → 课程生成成功 |
| Bilibili 视频导入 | 同上，覆盖 B 站特有差异 |
| 无字幕视频 | 优雅降级，明确报错 |
| 课程学习流程 | 进入课程 → 浏览列表 → 打开 section → 看内容 + 视频 |
| Mentor 对话 | 发消息 → SSE 流式回复 → 基于课程内容回答 |
| 苏格拉底模式 | mentor 在引导提问，非直接给答案 |
| 练习 | 生成 → 提交 → 评估反馈 |
| 间隔复习 | 学习后出现复习项 → 复习 → 更新间隔 |
| 记忆系统 | 多轮对话后 mentor 能引用之前的上下文 |

### LLM 后端兼容性测试

| 后端 | 测试重点 |
|---|---|
| Ollama（本地） | 零 API key 配置，模型下载引导，响应质量可接受 |
| OpenAI API | GPT-4o / GPT-4o-mini 全流程 |
| Anthropic API | Claude Sonnet 全流程 |
| DeepSeek API | 全流程，特别注意中文内容质量 |

### Docker 部署测试

| 场景 | 验证点 |
|---|---|
| 全新环境 docker compose up | 从零启动，所有服务健康 |
| 仅配置 Ollama | 最小配置路径跑通 |
| 数据库迁移 | Alembic 自动执行 |
| 服务重启 | 数据持久化，状态不丢失 |

### 自动化测试

| 层级 | 策略 |
|---|---|
| 后端单元测试 | pytest 补全关键路径覆盖 |
| API 集成测试 | 核心 endpoint 的 happy path |
| 前端冒烟测试 | 关键页面可渲染、不白屏 |
| E2E 冒烟测试 | 导入 → 学习 → 对话 → 练习 完整路径自动化 |

### 质量红线

- `docker compose up` 零报错启动
- 核心闭环手动走通
- 至少 2 个 LLM 后端验证通过
- 无安全漏洞暴露
- README 按步骤能跑通（找没接触过项目的人验证）
